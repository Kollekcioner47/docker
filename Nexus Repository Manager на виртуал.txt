Nexus Repository Manager на виртуальной машине с Docker. В этом руководстве мы установим его как обычное приложение на Ubuntu, чтобы он был полностью готов к работе с Docker-образами.

Шаг 0: Планирование и подготовка

Перед началом важно убедиться, что ваша виртуальная машина (ВМ) соответствует минимальным требованиям и правильно спланирована сеть.

*   **Системные требования**: Nexus — это Java-приложение, которое требует ресурсов.
    *   **Операционная система**: Ubuntu 20.04 или новее (также подойдут CentOS/RHEL 7+).
    *   **Процессор**: Минимум 2 ядра, рекомендуется 4.
    *   **Оперативная память**: Минимум 4GB, для production рекомендуется 8GB.
    *   **Дисковое пространство**: Минимум 20GB для работы, плюс место под ваши Docker-образы. Для серьезной нагрузки рекомендуется от 200GB.
    *   **Программное обеспечение**: Требуется Java (OpenJDK 17 или 21).

*   **Сетевое планирование**: Заранее определитесь с сетевыми портами. Стандартный веб-интерфейс Nexus работает на `8081`. Для Docker-репозитория нужно выбрать отдельный порт, например `8082`. Обязательно откройте эти порты в файрволе вашей ВМ.

Шаг 1: Установка и настройка Nexus

Мы установим Nexus из официального `.tar.gz` архива, настроим его как службу `systemd` и выделим для него отдельного пользователя для безопасности.

1.  **Подключитесь к ВМ** и выполните обновление системы:
    ```bash
    sudo apt update && sudo apt upgrade -y
    ```

2.  **Установите Java (OpenJDK 17)**:
    ```bash
    sudo apt install openjdk-17-jdk -y
    ```
    Проверьте установку: `java -version`.

3.  **Загрузите и распакуйте Nexus**:
    Перейдите в директорию `/opt` и загрузите последнюю версию Nexus OSS (ссылки лучше брать с [официального сайта](https://www.sonatype.com/products/sonatype-nexus-repository)):
    ```bash
    cd /opt
    sudo wget https://download.sonatype.com/nexus/3/nexus-3.75.1-01-unix.tar.gz
    sudo tar -xzvf nexus-3.75.1-01-unix.tar.gz
    ```
    Создайте символическую ссылку для удобства: `sudo ln -s /opt/nexus-3.75.1-01 /opt/nexus`.

4.  **Создайте пользователя `nexus`**:
    ```bash
    sudo useradd --system --no-create-home --shell /bin/bash nexus
    ```
    Назначьте права собственности на директории Nexus этому пользователю:
    ```bash
    sudo chown -R nexus:nexus /opt/nexus
    sudo chown -R nexus:nexus /opt/sonatype-work
    ```

5.  **Настройте память JVM (опционально, но важно)**:
    Отредактируйте файл `/opt/nexus/bin/nexus.vmoptions` и настройте параметры `-Xms` и `-Xmx`. Например, для сервера с 8GB RAM:
    ```bash
    -Xms2g
    -Xmx2g
    ```
    Это выделит Nexus 2GB оперативной памяти.

6.  **Создайте и запустите службу systemd**:
    Создайте файл сервиса `/etc/systemd/system/nexus.service`:
    ```ini
    [Unit]
    Description=Sonatype Nexus Repository Manager
    After=network.target

    [Service]
    Type=forking
    User=nexus
    Group=nexus
    ExecStart=/opt/nexus/bin/nexus start
    ExecStop=/opt/nexus/bin/nexus stop
    Restart=on-failure
    LimitNOFILE=65536

    [Install]
    WantedBy=multi-user.target
    ```
    После создания файла выполните:
    ```bash
    sudo systemctl daemon-reload
    sudo systemctl enable nexus
    sudo systemctl start nexus
    ```
    Nexus может запускаться пару минут. Следить за логами можно командой:
    ```bash
    sudo tail -f /opt/sonatype-work/nexus3/log/nexus.log
    ```
    Успешный запуск подтверждается логом `Started Sonatype Nexus OSS`.

Шаг 2: Первоначальная настройка Nexus

1.  **Войдите в веб-интерфейс**: Откройте браузер и перейдите по адресу `http://<IP-вашей-ВМ>:8081`.
2.  **Вход с правами администратора**: Нажмите "Sign In". Логин — `admin`. Пароль для первого входа находится в файле `/opt/sonatype-work/nexus3/admin.password`. Выполните команду на сервере, чтобы его посмотреть: `sudo cat /opt/sonatype-work/nexus3/admin.password`.
3.  **Следуйте инструкциям мастера настройки**: Система предложит сменить пароль администратора. Вы также можете разрешить анонимный доступ, если это соответствует вашим требованиям безопасности.

Шаг 3: Настройка Docker-репозитория в Nexus

Теперь создадим сам Docker-репозиторий.

1.  **Активируйте Docker Bearer Token Realm**:
    Это необходимо для аутентификации Docker-клиентов. В веб-интерфейсе Nexus перейдите в **Settings (шестеренка) → Security → Realms**. Перенесите "Docker Bearer Token Realm" из левой колонки в правую (Active) и нажмите "Save".

2.  **Создайте Blob Store (необязательно)**:
    Это хранилище для ваших Docker-образов. По умолчанию Nexus использует `default`, но вы можете создать отдельное. Перейдите в **Settings → Repository → Blob Stores** и нажмите "Create blob store". Выберите тип `File` и укажите путь, например, `/opt/nexus-blobs/docker`.

3.  **Создайте Docker Hosted Repository**:
    *   Перейдите в **Settings → Repository → Repositories** и нажмите "Create repository".
    *   Выберите тип `docker (hosted)`.
    *   В поле **Name** введите, например, `docker-hosted`.
    *   В разделе **HTTP** поставьте галочку и укажите **Port** `8082`. Это порт, через который Docker-клиент будет общаться с репозиторием.
    *   В выпадающем списке **Blob store** выберите хранилище, созданное на предыдущем шаге, или оставьте `default`.
    *   Остальные настройки можно оставить по умолчанию и нажать "Create repository".

Шаг 4: Настройка Docker-клиента

Чтобы ваш Docker-клиент мог работать с новым репозиторием, его нужно настроить.

1.  **Настройте Docker для работы с HTTP**:
    По умолчанию Docker требует HTTPS. Так как мы используем HTTP, нужно явно указать Docker'у доверять нашему репозиторию.
    *   Откройте файл конфигурации Docker: `sudo nano /etc/docker/daemon.json`.
    *   Если файл пуст или его нет, создайте его со следующим содержимым:
        ```json
        {
          "insecure-registries": ["<IP-вашей-ВМ>:8082"]
        }
        ```
        Если файл уже содержит другие настройки, просто добавьте эту строку.
    *   Сохраните файл и перезапустите Docker: `sudo systemctl restart docker`.

Шаг 5: Проверка (Push/Pull образа)

Всё готово, чтобы протестировать работу репозитория.

1.  **Аутентификация**: Войдите в репозиторий из Docker CLI на вашей клиентской машине:
    ```bash
    docker login <IP-вашей-ВМ>:8082
    ```
    Используйте логин `admin` и пароль, который вы задали при настройке Nexus.

2.  **Скачайте тестовый образ** (если его нет локально):
    ```bash
    docker pull hello-world
    ```

3.  **Подготовьте образ для загрузки в ваш репозиторий**:
    Docker-образ должен быть помечен (tag) адресом вашего репозитория.
    ```bash
    docker tag hello-world <IP-вашей-ВМ>:8082/hello-world
    ```

4.  **Загрузите образ в Nexus**:
    ```bash
    docker push <IP-вашей-ВМ>:8082/hello-world
    ```

5.  **Проверьте результат**: Откройте веб-интерфейс Nexus (`http://<IP-вашей-ВМ>:8081`), перейдите в **Settings → Repository → Repositories**, выберите ваш репозиторий `docker-hosted` и перейдите на вкладку **Browse**. Там вы должны увидеть загруженный образ.

6.  **Скачайте образ обратно для проверки**:
    ```bash
    docker pull <IP-вашей-ВМ>:8082/hello-world
    ```

(Опционально) Шаг 6: Настройка безопасности и Nginx

Для production-окружения настоятельно рекомендуется настроить HTTPS. Это можно сделать с помощью Nginx в качестве reverse proxy, который будет принимать HTTPS-запросы и перенаправлять их на HTTP-порт Nexus'а (8082).

Вот минимальная конфигурация Nginx для HTTPS:
```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /path/to/your/certificate.crt;
    ssl_certificate_key /path/to/your/private.key;

    location / {
        proxy_pass http://localhost:8082;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
Также можно использовать Nginx, чтобы скрыть внутренние порты Nexus, обслуживая всё через стандартные 80 и 443 порты.

### 📌 Заключение и важные замечания

1.  **Безопасность**: Никогда не используйте конфигурацию `insecure-registries` в production. Это руководство предназначено для ознакомления и тестирования. Для production-среды всегда настраивайте HTTPS.
2.  **Резервное копирование**: Регулярно создавайте резервные копии двух важнейших директорий Nexus: `/opt/sonatype-work/nexus3` (конфигурация, логи, метаданные) и хранилище blob-ов (путь к которому вы указали в Blob Store, например `/opt/nexus-blobs`).
3.  **Мониторинг**: Внимательно следите за использованием дискового пространства. По мере загрузки образов оно будет быстро заполняться. Также можно настроить политики удаления старых образов в самом Nexus.
4.  **Развитие инфраструктуры**: Nexus можно интегрировать в CI/CD пайплайны, например, с Jenkins. Это позволит автоматизировать сборку, тестирование и публикацию образов при каждом коммите в репозиторий.

Надеюсь, это руководство поможет вам успешно развернуть собственный Docker-репозиторий. Если у вас появятся вопросы на каком-либо из этапов, обращайтесь — я постараюсь помочь.
